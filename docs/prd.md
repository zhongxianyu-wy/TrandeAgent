## 2. 背景目标

### 2.1 背景

用户作为基金投资小白，面临"买什么/何时买/依据是什么/如何搭建策略/如何深度分析"五大痛点。同时是工程师，希望自建工具沉淀投资认知，而非依赖第三方推荐。需要把分析、择时、复盘的机械工作交给系统，关键结论主动推送到飞书。

### 2.2 目标

**业务目标：**
1. **3 个月内**搭建可运行的 MVP，每日盘后自动推送基金分析结论
2. **6 个月内**通过策略竞技场沉淀出 Top-5 个人策略组合
3. **12 个月内**形成可复用的基金认知体系（YAML 配置 + 历史回测 + 纸上模拟三件套）

**非业务目标（Out of Scope）：**
- 不做股票/可转债/期货
- 不做债基/货基/REITs
- 不做自动下单/对接券商
- 不做多用户支持
- 不做实时盘中行情
- 不做移动端 App / Web 前端

## 6. 详细功能描述

### F1：数据采集与缓存

**输入：** AkShare 全市场基金代码列表（~8000 只）
**处理：**
1. 拉取全市场基金基本信息（`fund_name_em`）
2. 增量拉取日频净值（`fund_open_fund_info_em`）
3. 拉取持仓、经理、规模等季度数据
4. 限流：1 req/s + UA 轮换
5. 失败重试 3 次 + 降级
6. 本地缓存：Parquet（净值）/ SQLite（元数据）
**输出：** `data/cache/` 目录，每日增量更新

**验收：**
- ✅ 首次全量拉取 ≤ 3 小时
- ✅ 增量更新 ≤ 20 分钟
- ✅ 单接口失败不影响整体流程
- ✅ 缓存命中率 > 95%

### F2：基金池筛选器

**输入：** `config/strategies.yaml` 筛选规则 + 全市场基金数据
**处理：**
1. 按 L1-L4 指标计算每只基金得分
2. 按规则过滤（规模 2-50 亿、经理任期 > 2 年、机构占比 > 30% 等）
3. 综合排序取 Top-N（默认 20）
**输出：** 候选基金列表（含得分 + 选中理由）

**规则示例（4433 法则 + 自定义）：**
```yaml
screener:
  - name: "近1年排名前1/4"
    field: "rank_1y_percentile"
    op: "<="
    value: 0.25
  - name: "规模2-50亿"
    field: "scale"
    op: "between"
    value: [2, 50]
  - name: "经理任期>2年"
    field: "manager_tenure_years"
    op: ">="
    value: 2
```

**验收：**
- ✅ 每日筛选结果写入飞书 Base "今日候选"视图
- ✅ 每只基金附"选中理由"（指标证据）
- ✅ 筛选规则变更后自动重算并提示影响范围

### F3：单基金深度分析

**输入：** 基金代码（用户主动触发，MVP 通过命令行/配置文件）
**处理：**
1. 拉取该基金 L1-L4 全部指标
2. 用 quantstats 生成业绩归因
3. LLM（DeepSeek-V3）把指标 JSON 翻译成中文报告
4. 强约束 prompt：每条结论必须引用数据，输出"建议/中性/回避"标签
5. 代码后校验：引用的指标必须存在于输入 JSON
**输出：** 中文 Markdown 报告（5 分钟可读懂）

**报告结构：**
1. 一句话结论 + 标签
2. 经理画像（任期、历史业绩、风格）
3. 业绩归因（收益率/同类排名/回撤/夏普）
4. 风格分析（持仓风格/行业集中度/换手率）
5. 现金流（份额变动/机构持有变化）
6. 风险提示
7. 是否建议进入观察池

**验收：**
- ✅ 报告每条结论都有"【依据：<指标名>=<数值>】"
- ✅ 标签明确三选一
- ✅ LLM 不得编造未提供的字段（后校验拦截）

### F4：择时信号引擎

**输入：** 观察池基金列表 + 信号规则
**处理：**
1. 计算技术面信号（MA 金叉死叉、MACD、RSI、布林带）
2. 计算基本面信号（PE/PB 分位、股债收益差）
3. 计算基金专属信号（回撤补仓、定投止盈、大跌预警）
4. 综合给出四档信号：加仓/持有/减仓/止损
5. 每个信号附"为什么"（触发规则的指标值）
**输出：** 信号列表 + 卡片消息

**信号示例：**
```
基金：华夏成长混合（000001）
信号：加仓 ⬆️
理由：
  【依据：PE分位=15% < 20% 阈值】触发低估加仓
  【依据：回撤=-8% > -10% 止损线】无止损风险
  【依据：MA200向上】长期趋势向上
```

**验收：**
- ✅ 信号必须可解释（附触发规则的指标值）
- ✅ 大跌（单日 < -3%）即时单独推送
- ✅ 信号写入飞书 Base "信号"表

### F5：策略模拟竞技场（核心创新）

**输入：** 15 个策略原型 + 8 位大师心智模型 + 差异维度矩阵
**处理：**

**子功能 F5.1：策略生成器**
1. 加载 15 个策略原型（4433/双动量/网格/定投/风险平价...）
2. 加载 8 位大师心智模型档案
3. LLM 在差异维度矩阵（投资域×择时×频率×风控×集中度 = 2048 组合）约束下生成 50-200 个差异化策略
4. 每个策略可追溯到来源（"巴菲特护城河 × 双动量滤波"）
5. 去重检查：避免同质化

**子功能 F5.2：历史回测**
1. 用 VectorBT 对 50-200 策略 × 3-5 年历史数据并行回测
2. 计算每策略：年化收益、夏普、最大回撤、胜率
3. 对胜出的 Top-20 用 Backtrader 做精细回测（含手续费/滑点）

**子功能 F5.3：纸上模拟**
1. 策略上线后每日用真实行情记录收益
2. 至少跑满 30 个交易日才能进 Top-5
3. 双轨交叉验证：回测结果 vs 纸上模拟，差异 > 20% 标记可疑

**子功能 F5.4：领域排名**
1. 按投资风格领域分组（价值/成长/红利/趋势/逆向/全球配置/指数增强/低波 = 8 领域）
2. 每领域综合得分 = 收益×0.5 + 夏普×0.3 + 回撤倒数×0.2
3. 取每领域 Top-5
4. 写入飞书 Base "策略竞技场"表

**输出：** 飞书 Base 多视图（按领域/按收益/按夏普）

**验收：**
- ✅ 策略可追溯到来源（原型 + 大师 + 参数）
- ✅ 双轨结果同时可见
- ✅ 排名按领域分组，每领域 Top-5
- ✅ 用户可一键"采用某策略"→ 进入正式观察池择时通道
- ✅ 策略总数 50-200，不超过 200（R7 风险缓解）

### F6：飞书单聊推送

**输入：** 当日基金池/信号/竞技场结果
**处理：**
1. 渲染卡片 JSON 模板
2. 通过 `lark-cli im +messages-send --as bot` 推送
**输出：** 飞书单聊卡片消息

**卡片内容：**
- 今日组合表现（观察池平均收益）
- 观察池异动（涨跌幅 Top-3）
- 触发的信号（加仓/减仓/止损）
- 候选新基金（Top-3）
- 竞技场亮点（当日 Top-1 策略）
- 按钮：跳转飞书 Base 详细视图

**验收：**
- ✅ 每个交易日 16:00 推送（节假日不发）
- ✅ 卡片可点击跳转飞书 Base
- ✅ 推送失败有重试 + 日志告警

### F7：飞书多维表格

**Schema：**
- **基金池表**：代码/名称/类型/规模/经理/夏普/回撤/选中理由/是否观察池
- **信号表**：日期/基金代码/信号/理由/触发指标
- **策略竞技场表**：策略ID/来源/领域/年化收益/夏普/回撤/纸上模拟收益/综合得分/排名
- **复盘表**：日期/决策/结果/反思

**验收：**
- ✅ 每日批量写入（≤200 条/批）
- ✅ 字段类型正确（数字/百分比/日期/单选）
- ✅ 视图：今日候选 / 观察池 / 信号 / 竞技场 Top-5

### F8：YAML 策略配置

**输入：** 用户编辑的 YAML 配置
**处理：**
1. Pydantic 校验配置 schema
2. 变更后自动重新计算信号并提示影响范围
3. Git 版本管理
**输出：** 配置生效 + 影响范围报告

**配置示例：**
```yaml
# config/strategies.yaml
observation_pool:
  - "000001"  # 华夏成长
  - "161725"  # 招商中证白酒

screener_rules:
  scale: [2, 50]  # 亿
  manager_tenure_years: 2
  institution_holding_ratio: 0.3

signal_rules:
  pe_percentile_buy: 0.2   # PE 分位 < 20% 加仓
  pe_percentile_sell: 0.8  # PE 分位 > 80% 减仓
  drawdown_stop: -0.15     # 回撤 > 15% 止损
  daily_drop_alert: -0.03  # 单日跌 > 3% 预警

arena:
  strategy_count: 100      # 生成 100 个策略
  backtest_years: 5
  forward_min_days: 30
  domains: [价值, 成长, 红利, 趋势, 逆向, 全球配置, 指数增强, 低波]
```

**验收：**
- ✅ 改配置后系统自动重算
- ✅ 提示影响范围（"本次修改影响 3 只基金的信号"）
- ✅ 配置可回滚（Git）

### F9：定时调度

**输入：** macOS launchd plist

**验收：**
- ✅ 16:00 自动运行（±5 分钟）
- ✅ 节假日不发
- ✅ 关机后开机自动补发漏推送

### F11：本地后端 API（FastAPI）— v0.3 新增

**输入：** 业务模块（src.data/src.indicators/src.screener/src.analyzer/src.signal/src.arena）
**处理：**
1. FastAPI 应用直接 import 业务模块，不重复实现逻辑
2. Pydantic 定义所有响应 schema
3. 自动生成 OpenAPI 文档（`/docs`）
4. 提供 RESTful API：
   - `GET /api/funds` 基金列表（含筛选）
   - `GET /api/funds/{code}` 单基金详情（含 L1-L4 指标）
   - `GET /api/funds/{code}/report` 单基金 LLM 报告
   - `POST /api/funds/{code}/analyze` 触发 AI 重新分析
   - `GET /api/strategies` 策略列表（含领域分组+排名）
   - `GET /api/strategies/{id}` 策略详情 + 周期分析数据
   - `POST /api/strategies/{id}/adopt` 采用策略
   - `GET /api/discover?domain=&window=` 发现页推荐
   - `GET /api/observation` 观察池
   - `POST /api/observation/{code}` 加入观察池
   - `GET/POST/PUT /api/config` 配置管理
   - `POST /api/jobs/{type}` 手动触发任务（拉数据/回测/分析）
5. CORS 允许 localhost:3000
**输出：** OpenAPI JSON + REST API

**验收：**
- ✅ 所有 API 返回 Pydantic 校验过的响应
- ✅ `/docs` 可交互文档可用
- ✅ 单基金分析 API ≤ 30s 返回
- ✅ 策略列表 API ≤ 2s 返回（含 100 策略 × 8 领域）
- ✅ CORS 配置正确（前端可调）

## 7. 非功能需求（NFRs）

| 维度 | 要求 |
|---|---|
| **性能** | 单日全流程 ≤ 30 分钟（拉数+计算+推送）；VectorBT 参数扫描 ≤ 10 分钟 |
| **可用性** | 单接口失败不影响整体流程；缓存命中率 > 95% |
| **可靠性** | 推送失败重试 3 次；日志告警 |
| **成本** | 月度 LLM 成本 ≤ ¥10（DeepSeek-V3）；零云成本（本地 Mac） |
| **安全** | 个人持仓数据不出本地；写入飞书的数据脱敏（不写绝对金额，只写比例） |
| **隐私** | 飞书数据脱敏（不写绝对金额）；API Key 用环境变量 |
| **可维护性** | 模块化设计；Pydantic 配置校验；pytest 覆盖率 > 70% |
| **可移植性** | 抽象 DataProvider/LLMClient/FeishuClient 接口，便于迁云 |
| **合规** | AkShare 数据仅个人决策；不分发；LLM 不得编造（后校验拦截） |

## 8. 交互设计（UI/UX）

### 8.1 飞书单聊卡片

### 8.2 飞书多维表格视图

- **今日候选**：筛选 = 当日新增；字段 = 代码/名称/选中理由/综合得分
- **观察池**：筛选 = is_observation=true；字段 = 代码/当前信号/累计收益
- **信号**：日期/基金/信号档位/理由
- **竞技场 Top-5**：按领域分组，Top-5 综合得分

## 9. 验收标准（AC）

> 详细 AC 见各功能 §6 验收点。此处列端到端验收场景。

### AC-1：每日盘后推送 E2E
**Given** 一个交易日 16:00
**When** launchd 触发 main.py
**Then**
1. 拉取当日全市场基金数据（≤ 20 分钟）
2. 计算筛选/信号/竞技场（≤ 5 分钟）
3. 写入飞书 Base 4 张表
4. 推送单聊卡片（≤ 5 分钟内到达）
5. 总耗时 ≤ 30 分钟

### AC-2：单基金分析 E2E
**Given** 用户配置 `analyze: ["000001"]`
**When** 触发分析
**Then**
1. 5 分钟内生成中文报告
2. 报告每条结论都有"【依据：...】"
3. 报告结尾有"建议/中性/回避"标签
4. 后校验通过（无幻觉）

### AC-3：策略竞技场 E2E
**Given** 首次启动
**When** 触发策略生成
**Then**
1. 生成 50-200 个差异化策略（可追溯来源）
2. 历史回测 ≤ 10 分钟（VectorBT）
3. Top-20 精细回测（Backtrader，含手续费）
4. 写入飞书 Base "策略竞技场"表
5. 8 领域各取 Top-5

### AC-4：节假日不发
**Given** 国庆节（10/1-10/7）
**When** launchd 触发
**Then** main.py 启动后 chinese_calendar 判定为非交易日，直接 return

### AC-5：配置变更影响提示
**Given** 用户修改 `strategies.yaml`
**When** 提交配置
**Then** 系统输出"本次修改影响 N 只基金的信号"，并重新计算
