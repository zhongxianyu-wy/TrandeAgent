# 前端设计文档（飞书侧 UI 统一规范）

> **Step 7 产出**
> 本项目 MVP 阶段无独立 Web/App 前端（见 [context.md §4.2](../docs/context.md)），所有 UI 通过飞书侧呈现：
> 1. **单聊卡片**（Interactive Card）
> 2. **多维表格视图**（Base Views）
>
> 本文档统一规范所有 feature 的 UI 设计，供 #2/#5/#6/#7/#8 实现。

---

## 1. 设计原则

| 原则 | 说明 |
|---|---|
| **信息密度优先** | 用户每日看 5-10 分钟，单卡片必须承载高密度信息 |
| **可点击下钻** | 卡片永远是"概览 + 跳转 Base 详情" |
| **数据可追溯** | 每个数字附"依据"，对应 research.md 防幻觉原则 |
| **视觉一致** | 统一颜色编码（绿=加仓/红=减仓/灰=持有/橙=止损） |
| **移动端友好** | 飞书单聊 70% 在手机查看，卡片宽度自适应 |

```
│ └─────────────────────────────────────────┘│
│                                              │
│ 📈 今日信号                                  │
│  🟢 加仓  华夏成长（000001）                 │
│        PE分位 15% < 20%，触发低估加仓        │

│  2. 兴全合宜（163417）  经理任期 5.2年       │
│  3. 易方达蓝筹（005827）规模 28亿            │
│                                              │

│  今日 Top-1: 巴菲特护城河×双动量             │

│                                              │
│ [📋 查看完整 Base]  [⚙️ 管理观察池]          │  <- action buttons
└─────────────────────────────────────────────┘
```

**JSON 结构要点（#2 feishu-io cards.py 实现）：**
- `header.template: "blue"` + 标题
- `elements` 含 4 个 markdown 块（组合/信号/候选/竞技场）
- `action` 块含 2 个 button（open_url 跳转 Base）

**视觉规范：**
- 信号档位 emoji：🟢加仓 / 🔴减仓 / ⚫持有 / 🟠止损
- 数据条形：用 column_set 模拟进度条
- 颜色：正值绿、负值红（A 股惯例，与美股相反）

## 4. 多维表格视图设计

### 4.1 总体 schema

| 表名 | 用途 | 主要字段 |
|---|---|---|
| 基金池 | 全部基金 + 候选 + 观察池 | code, name, type, scale, sharpe, drawdown, reason, is_observation |
| 信号 | 每日信号记录 | date, fund_code, level, reasons, score |
| 策略竞技场 | 策略 + 排名 | strategy_id, prototype, mind_model, domain, return, sharpe, score, rank |
| 复盘 | 用户决策记录 | date, decision, fund_code, outcome, reflection |

### 4.2 视图："今日候选"（#5 fund-screener）

- **筛选：** `created_at = 今天` AND `is_observation = false`
- **排序：** 综合得分 降序
- **字段可见：** fund_code, fund_name, fund_type, scale, sharpe, reason, + "加入观察池"按钮（公式字段生成链接）

### 4.3 视图："信号"（#7 signal-engine）

- **筛选：** 用户可选"今日"/"本周"/"全部"
- **分组：** 按 level（加仓/减仓/持有/止损）
- **排序：** score 降序
- **字段：** date, fund_code, fund_name, level, reasons

### 4.4 视图："竞技场 Top-5"（#8 strategy-arena）

- **筛选：** `is_qualified = true`（满 30 天纸上模拟）
- **分组：** 按 domain（8 领域）
- **排序：** 组内 composite_score 降序
- **限制：** 每组只显示前 5（用视图筛选公式）
- **字段：** strategy_id, prototype, mind_model, return_backtest, return_forward, sharpe, drawdown, composite_score, rank

## 6. 响应式与移动端

飞书卡片自适应宽度。设计时：
- 单行字符数 ≤ 30（手机可读）
- 多列用 `column_set`，手机自动堆叠
- 按钮区不超过 3 个，避免手机溢出
