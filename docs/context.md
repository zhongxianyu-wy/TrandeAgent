# Context.md — 需求上下文

> 本文档由 Step 1 (superpowers brainstorming + mattpocock /grill-me) 产出，作为后续 PRD、Spec、技术调研、架构选型的唯一需求源。
> 任何 PRD/Spec 中出现与本文档冲突的需求，以本文档为准；如需变更，先更新本文档并经用户确认。

---

## 1. 项目背景 (Why)

用户（项目所有者，下称"用户"）当前在做一个**个人基金投资决策**场景：
- 作为基金投资小白，对「买什么 / 何时买 / 买入依据 / 策略搭建 / 基金深度分析」均不熟悉；
- 同时也是一名工程师，希望通过自建工具沉淀自己的投资认知，而不是依赖第三方推荐；
- 希望把分析、择时、复盘等机械性工作交给系统，自己只做最终决策；
- 日常工作高强度，需要把关键结论**主动推送**到他常用的协作工具（飞书），而不是被动去查询。

不涉及股票交易，**只覆盖公募基金**（含 QDII、场外指数/ETF 联接、主动股票基金）。

| 容错底线 | 亏损容忍中等；无法接受"踩雷基金"（规模过小、风格漂移、风格激进） |

## 4. 核心需求 (What) — grill-me 已澄清

### 4.1 已确定的需求边界

| 决策点 | 用户选择 | 说明 |
|---|---|---|
| **基金数据源** | AkShare / Tushare | 开源聚合，覆盖天天基金 + 东方财富数据；零成本；需自己处理反爬 |
| **飞书接入形态** | ① 机器人单聊推送<br>② 多维表格存储 | 不做群聊 @bot 实时交互（MVP 阶段），不做云文档报告 |
| **策略深度** | AI 辅助决策 | LLM 结合多维数据给出"推荐理由 + 买入时间点建议"，**最终决策由用户确认**，不做自动跟单 |
| **基金类型范围** | ① QDII<br>② 场外指数 / ETF 联接<br>③ 主动股票基金 | **不包含债基、货基**（用户已确认） |
| **推送频率** | 每日盘后 | A 股收盘后（15:00 后）固定时点推送，周末/节假日不发 |
| **MVP 范围** | 完整 MVP | 基金池筛选 + 单基金 AI 分析 + 择时信号 + **策略模拟竞技场** 首版全打通 |
| **运行环境** | 本地 Mac | 关机不推送；后续可迁云（架构需保留迁移空间） |
| **技术栈** | 待调研 | Step 2 派调研 Agent 对比后给出 |
| **仓库名** | TrandeAgent | 与目录同名 |
| **GitHub 远程** | 用户提供现有仓库 URL | 用户后续提供，先本地 init |
| **模拟策略数量** | **50–200 个**（v0.2 新增） | 用户明确不要凑数 10000，遵循业内主流量级（如万得策略指数、聚宽策略广场）。策略空间由「主策略原型 × 关键参数档位」组成 |
| **模拟形态** | **双轨并行**（v0.2 新增） | ① 历史回测（3–5 年）：一次性跑完看哪些策略在历史中胜出；② 纸上模拟（forward）：策略上线后每日用真实行情记录；两轨结果交叉验证一致性 |
| **心智模型范围** | **精选 5–8 位投资大师**（v0.2 新增） | 用户选择"深度总结每人心智模型，不做面上广撒网"。候选名单由调研 Agent 给出并经用户确认 |
| **策略排名输出** | **各领域 Top-5**（v0.2 新增） | "领域"= 投资风格领域（如价值/成长/红利/趋势/逆向/全球配置/指数增强…），不是行业。每个领域取收益/夏普综合排名前 5 |
| **Mac 端前端形态** | **Next.js Web 应用**（v0.3 新增） | 浏览器访问 localhost，不起桌面壳（不做 Tauri/Electron）。日常深度查看/管理在前端，关键信号/报告走飞书 |
| **前后端架构** | **FastAPI 本地后端 + Next.js 前端**（v0.3 新增） | FastAPI 直接 import 业务模块（src.data/src.indicators...），Next.js 通过 TanStack Query 调用；Pydantic 自动生成 OpenAPI → 前端类型安全 |
| **前端交互深度** | **完整管理台**（v0.3 新增） | 不仅是 Dashboard：可编辑 YAML 规则、管理观察池、采用/停用策略、触发分析、版本回滚 |
| **前后端启动方式** | **一键脚本**（v0.3 新增） | 双击 .command 或 `./start.sh`，同时拉起 uvicorn + Next.js dev server；可选打包桌面快捷方式 |
| **周期分析图表** | 多周期收益柱状图 + 净值曲线+回撤+基准 + 现金流时序图（v0.3 新增） | 用户明确选择这三类，**不含**月度热力图 |
| **视觉风格** | **现代 SaaS 风**（v0.3 新增） | 类似 Notion/Linear/Stripe Dashboard：干净中性、阅读友好、信息密度高、不追求"专业金融感"的黑色界面 |
| **前端与飞书关系** | **分工并重**（v0.3 新增） | 前端 = 日常深度查看 + 完整管理；飞书 = 关键信号主动推送 + 报告卡片 + Base 数据存储。两者不替代，互补 |

## 5. 关键用户故事 (User Stories)

### US-1：基金池筛选（对应 P1）
> 作为用户，我希望系统每天盘后能从全市场（QDII / 场外指数 / ETF 联接 / 主动股基）中，按我配置的规则（最大回撤、夏普、规模、基金经理任期、机构持仓比等）筛选出 Top-N 候选基金，并写入飞书多维表格，这样我就不用再被天天基金排行榜洗脑。

**验收点：** 我能在飞书 Base 里看到一个"今日候选"视图，每只基金附带为什么被选中的指标证据。

### US-2：单基金深度分析（对应 P2）
> 作为用户，当我听说一只基金时，我希望能用基金代码或名称一键触发深度分析，5 分钟内在飞书单聊里收到一份结构化报告：经理画像 / 持仓风格 / 业绩归因 / 风险指标 / 同类排名 / 现金流（申购赎回趋势）/ 是否值得进观察池。

**验收点：** 报告每条结论都有数据支撑，并给出明确"建议/中性/回避"标签。

### US-3：择时信号（对应 P3、P6）
> 作为用户，我希望对观察池中的基金，系统每天盘后基于技术面（均线、PE 分位、回撤幅度）+ 基本面信号，给出"加仓 / 持有 / 减仓 / 触发止损"四档信号，并解释理由；大跌（单日跌幅 >X%）时即时单独推送。

**验收点：** 信号必须可解释（给出触发规则的指标值），不能是黑盒。

### US-4：每日盘后推送（对应 P6）
> 作为用户，我希望每个交易日 16:00（默认可配置）在飞书单聊收到一条卡片消息，包含：今日组合表现 / 观察池异动 / 触发的信号 / 候选新基金。

**验收点：** 卡片可点击跳转到飞书 Base 详细视图。

### US-5：策略可视化沉淀（对应 P4）
> 作为用户，我希望把我的筛选规则、择时规则、风险阈值集中存到一个 YAML 配置文件里，版本管理，能随时回滚。

**验收点：** 改配置后系统自动重新计算信号并提示影响范围。

### US-6：策略模拟竞技场（v0.2 新增 — 对应 P1+P4）
> 作为用户，我希望系统能基于"主流策略原型 + 5–8 位投资大师心智模型"自动生成 50–200 个差异化策略，每天用真实行情双轨模拟（历史回测 + 纸上模拟），并在飞书 Base 里给出**每个投资风格领域（价值/成长/红利/趋势/逆向/全球配置/指数增强）的 Top-5 策略排名**，这样我就不用凭直觉选策略，而是看竞技结果说话。

**验收点：**
1. 策略可追溯到来源（"来自巴菲特护城河模型 × 双动量滤波"）；
2. 双轨结果同时可见（历史年化收益 + 当月纸上模拟收益），便于交叉验证；
3. 排名按"领域 + 综合得分（收益×0.5 + 夏普×0.3 + 回撤倒数×0.2）"分组；
4. 用户可一键"采用某策略"→ 该策略进入正式观察池择时信号通道。

### US-7：Mac 端策略池总览与周期分析（v0.3 新增 — 对应 P1+P4）
> 作为用户，我希望在 Mac 浏览器打开 localhost，看到一个现代 SaaS 风的策略池总览页，能按投资领域（价值/成长/红利/趋势/逆向/全球配置/指数增强/低波）切换查看每个策略的：基本信息、来源原型+心智模型、历史回测收益、当月纸上模拟收益、综合得分、领域排名。点击某个策略能进入该策略的**周期分析详情**，看多周期收益柱状图（日/周/月/季/年）+ 净值曲线（含回撤阴影+沪深300基准对比）+ 现金流时序图，一眼看懂策略在不同时间尺度的表现。

**验收点：**
1. 策略池总览页按 8 领域分组，每组显示 Top-5 + 全部策略列表；
2. 策略详情页含 3 类图表（多周期收益柱状图、净值曲线+回撤+基准、现金流时序图）；
3. 图表数据可下钻（点击某月柱子看当月明细）；
4. 支持时间范围筛选（近 1 月/3 月/6 月/1 年/3 年/全部）。

### US-8：Mac 端单基金深度解读（v0.3 新增 — 对应 P2）
> 作为用户，当我在策略池或候选池看到某只基金时，希望点击进入该基金的**深度解读页**，比飞书卡片更详细：经理画像时间轴、持仓行业雷达图、业绩归因对比表（vs 同类平均/沪深300）、风险指标仪表盘、现金流瀑布图、LLM 报告（含可折叠的"原文指标 JSON"区），并能在此页一键"加入观察池"或"触发 AI 重新分析"。

**验收点：**
1. 单基金页含 ≥ 6 个可视化模块（经理/持仓/业绩/风险/现金流/LLM 报告）；
2. 业绩归因含 3 列对比（本基金/同类平均/沪深300）；
3. LLM 报告每条结论的【依据】可点击跳转到对应指标；
4. "触发 AI 重新分析"按钮调 FastAPI 后端，返回新报告（≤ 30s）；
5. 页面所有数字与 SQLite/Parquet 缓存一致（防数据漂移）。

### US-9：Mac 端各领域基金推荐（v0.3 新增 — 对应 P1）
> 作为用户，我希望有一个"发现"页，按投资领域（价值/成长/红利/趋势/逆向/全球配置/指数增强/低波）展示**今日推荐基金**，每个领域 Top-5，每只基金附推荐理由（命中的筛选规则+指标值）。能切换"今日/本周/本月"时间窗，并能一键加入观察池。

**验收点：**
1. 发现页 8 领域 × Top-5 推荐基金；
2. 每只基金附推荐理由（命中规则+指标证据）；
3. 时间窗切换（今日/本周/本月）；
4. 一键"加入观察池"→ 写入 feishu Base + 本地 SQLite。

### US-10：Mac 端完整管理台（v0.3 新增 — 对应 P4+P5）
> 作为用户，我希望前端不只是看板，而是完整管理台：能可视化编辑筛选规则/择时规则（YAML 表单化）、管理观察池（增删/排序）、采用或停用竞技场策略、手动触发回测/分析、查看配置版本历史并一键回滚。

**验收点：**
1. YAML 规则表单化编辑（不用手写 YAML）；
2. 保存配置时显示"影响范围"（影响 N 只基金信号）；
3. 配置变更自动 git commit，前端可看历史 + 回滚；
4. 策略采用/停用按钮即时生效（写入 SQLite）；
5. 手动触发：拉数据/跑回测/生成报告，前端显示进度条。

## 6. 关键约束 (Constraints)

| 维度 | 约束 |
|---|---|
| **数据合规** | AkShare 抓取的数据仅用于个人决策，不对外提供服务，不做二次分发 |
| **反爬** | 必须有请求限流（建议 ≥1 req/s 间隔）和 User-Agent 轮换；失败要降级而非崩溃 |
| **数据时效** | 日频数据 T+1 收盘后可用，不接受实时性要求 |
| **飞书配额** | 单聊机器人消息有 QPS 限制（5 msg/s），多维表格 API 也有限制，需批量写 |
| **AI 成本** | LLM 调用需可控成本，单基金分析控制在 ~3K tokens 以内 |
| **本地运行** | 必须能在 macOS 本地零成本运行；不能强制依赖云服务（除 LLM API 外） |
| **隐私** | 个人持仓数据不出本地；写入飞书的数据需脱敏（不写绝对金额，只写比例） |

## 7. 关键风险 (Risks)

| # | 风险 | 等级 | 缓解措施 |
|---|---|---|---|
| R1 | AkShare 接口不稳定 / 字段变更 | 高 | 封装数据访问层，单元测试 + 多数据源降级（Tushare） |
| R2 | 反爬被封 IP | 中 | 限流 + 缓存 + 失败重试 |
| R3 | LLM 幻觉给出错误推荐 | 高 | 强制 RAG（结构化指标先于 LLM 推理），输出必须引用数据 |
| R4 | 飞书多维表格字段设计不合理导致后期重构 | 中 | Step 5 Spec 阶段先冻结 schema |
| R5 | 用户对指标理解偏差导致策略误用 | 中 | 每个信号必须附"为什么" |
| R6 | 本地 Mac 关机导致漏推送 | 低 | 启动时检测漏推送窗口，补发 |
| R7 | **10000 策略组合爆炸导致计算/存储失控** | 高 | v0.2 已澄清为 50–200 个，用「主策略原型 × 参数档位」约束；超过 200 需用户确认 |
| R8 | 回测过拟合：历史表现好不代表未来好 | 高 | 双轨并行（回测+纸上模拟）交叉验证；纸上模拟至少跑满 30 个交易日才能进 Top-5 |
| R9 | LLM 生成 50–200 个策略时大量同质化 | 中 | 用"差异维度矩阵"（投资域 / 择时逻辑 / 风控阈值 / 持仓集中度）强制差异化 |
| R10 | 投资大师心智模型被 LLM 拟人化扭曲 | 中 | 仅取公开著作原话（年报/股东信/书籍）作 ground truth，prompt 强约束不得发挥 |
| R11 | **前后端类型漂移**：FastAPI 改 schema 后前端未同步，运行时崩溃 | 中 | Pydantic 自动生成 OpenAPI → openapi-typescript 自动生成 TS 类型，CI 校验 |
| R12 | **本地前后端启动复杂度**：用户难一次跑通 uvicorn+next | 高 | 一键脚本 + .command 双击启动 + 启动诊断（端口/依赖/数据缓存检查） |
| R13 | **图表性能**：5 年日频净值 × 多基金对比可能卡顿 | 中 | ECharts 大数据优化（dataSampling）+ 后端预聚合 + 前端虚拟化 |

## 8. 仍待 Step 2 调研确认的问题 (Open Questions for Research)

> 这些问题不阻塞 PRD 撰写，但必须在架构选型前由调研 Agent 给出证据。

1. **AkShare vs Tushare 数据完整性对比**：基金经理任期、机构持仓比例、申赎现金流这些字段，哪个源覆盖更全？

## 9. 名词表 (Glossary)

| 术语 | 含义 |
|---|---|
| 观察池 | 用户标记为"重点关注"的基金集合，写入飞书 Base |
