# Feature #5 — fund-screener（基金池筛选器）

> 来源：[PRD §6 F2](../docs/prd.md)

---

## 1. Feature 简介

按用户配置的 YAML 规则（4433 法则 + 自定义），对全市场基金（来自 #1 + #4）综合打分排序，输出 Top-N 候选基金，写入飞书 Base。

---

## 2. 用户故事

- **作为用户**，我希望每日盘后看到 Top-20 候选基金，含选中理由，不必再被天天基金排行榜洗脑

---

## 3. 输入 / 输出
- 输入：`config/strategies.yaml` 筛选规则 + 全市场指标（来自 #4）
- 输出：候选基金列表（含综合得分 + 选中理由）→ 写入飞书 Base "基金池"表

---

## 4. 功能需求（FR）

### FR-1：规则引擎
- 支持操作符：`>=` `<=` `between` `in` `percentile_top`
- 支持 AND 组合
- 规则示例：4433 法则 + 规模 2-50 亿 + 经理任期 > 2 年 + 机构占比 > 30%

### FR-2：综合打分
- 每条规则匹配得 1 分
- 多维度加权（用户可配置 weights）

### FR-3：选中理由
- 每只候选基金附"为什么被选中"（命中的规则列表 + 指标值）

### FR-4：写入飞书 Base
- 每日清空"今日候选"视图 → 写入新一批

### FR-5：手动触发
- `python -m src.screener run --top 20`
- `python -m src.screener run --rules custom.yaml`

---

## 5. 非功能需求
- 性能：全市场筛选 ≤ 30s
- 可配置：规则全部 YAML 化

---

## 6. 验收标准（AC）

### AC-1：4433 法则
**Given** 全市场 + 4433 规则
**When** 运行
**Then**
1. 返回 Top-20
2. 每只基金命中至少 3 条规则
3. 附选中理由

### AC-2：写入 Base
**Given** 筛选结果
**When** 写入飞书 Base
**Then**
1. "今日候选"视图刷新
2. 每条记录有 reason 字段

### AC-3：自定义规则
**Given** 用户改 YAML
**When** 重新运行
**Then** 按新规则筛选

---
