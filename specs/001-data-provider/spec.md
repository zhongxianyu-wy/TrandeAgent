## 1. Feature 简介

**一句话：** 提供一个统一的数据访问层，封装全市场公募基金（QDII / 场外指数 / ETF 联接 / 主动股票基金）的日频数据采集、本地缓存、限流与失败降级，作为下游所有业务模块（筛选、分析、信号、竞技场）的唯一数据来源。

---

## 2. 用户故事（业务价值）

- **作为系统**（下游模块），我需要一个统一的数据接口，这样我不必关心数据从哪里来、缓存到哪、如何限流
- **作为用户**，我希望首次全量拉取 ≤ 3 小时、每日增量更新 ≤ 20 分钟，这样系统在 16:00 推送前能跑完全流程
- **作为用户**，我希望某个上游接口临时挂掉时，系统仍能用缓存数据继续运行（只是当天数据略旧），而不是整体崩溃

---

## 3. 输入 / 输出契约

### 3.1 输入

| 输入 | 来源 | 说明 |
|---|---|---|
| 全市场基金代码列表 | 上游公开数据 | ~8000 只公募基金 |
| 基金类型过滤 | 调用方 | 仅 QDII / 场外指数 / ETF 联接 / 主动股票基金（不含债基/货基，见 context.md） |
| 数据日期范围 | 调用方 | 如 "近 5 年" / "2020-01-01 至今" |
| 字段需求清单 | 调用方 | L1 基本面 / L2 业绩 / L3 风格 / L4 现金流（详见 research.md §4.1） |

### 3.2 输出

| 输出 | 形态 | 说明 |
|---|---|---|
| 基金基本信息 | DataFrame | 代码/名称/类型/经理/规模/费率/成立日 |
| 日频净值序列 | DataFrame | 日期/单位净值/累计净值/涨跌幅 |
| 持仓明细（季频） | DataFrame | 季末日期/重仓股/行业配置 |
| 经理信息 | DataFrame | 姓名/任期起止/历史业绩/管理规模 |
| 份额变动（季频） | DataFrame | 季末份额/机构持有比例 |
| 数据新鲜度报告 | dict | 各字段的最近更新时间、缓存命中标记、失败原因 |

## 4. 功能需求（FR）

### FR-1：全市场基金代码列表拉取
- 必须返回全市场公募基金代码
- 必须能按类型过滤（QDII / 场外指数 / ETF 联接 / 主动股票基金）
- **显式排除**债基、货基（按 context.md 边界）

### FR-2：日频净值增量拉取
- 每日盘后（T+1 收盘后可用）增量拉取
- 支持首次全量历史回填（最多回填 5 年）
- 必须处理拆分/折算（前复权）

### FR-3：季度数据拉取
- 持仓明细：每季度末后 15 个工作日内更新
- 经理信息：任期变动检测
- 份额变动：机构持有比例变化

### FR-4：本地缓存
- 净值数据按"代码+日期"缓存，不重复拉取
- 元数据（基本信息）按代码缓存，每周刷新一次
- 缓存命中时直接返回，不发起网络请求

### FR-5：限流
- 默认 1 req/s 速率限制
- User-Agent 轮换
- 并发请求 ≤ 1（避免触发上游封禁）

### FR-6：失败处理
- 单接口失败重试 3 次（指数退避：1s / 2s / 4s）
- 重试失败后用缓存数据降级，标记 `is_stale=True`
- 批量任务中单只基金失败不影响其他基金

### FR-7：数据新鲜度报告
- 每次拉取后输出 `data_freshness.json`：
  - 每只基金每字段的最近更新时间
  - 哪些字段命中缓存
  - 哪些字段失败及原因

## 5. 非功能需求（NFR）

| 维度 | 要求 |
|---|---|
| 性能 | 首次全量 ≤ 3 小时；增量 ≤ 20 分钟 |
| 可靠性 | 单接口失败不阻塞整体；缓存命中率 > 95% |
| 可观测 | 每次拉取产出新鲜度报告；日志含限流次数/重试次数/失败原因 |
| 幂等性 | 同一天重复运行不产生副作用（增量按日期去重） |
| 数据完整性 | 净值数据缺失天数 < 1%（剔除非交易日）；字段缺失率 < 5% |
| 隐私 | 不涉及个人数据，无隐私要求 |

## 6. 验收标准（AC）

### AC-1：首次全量拉取
**Given** 空缓存
**When** 触发首次全量拉取（5 年历史）
**Then**
1. 总耗时 ≤ 3 小时
2. 8000 只基金中 ≥ 95% 有完整净值数据
3. 输出 `data_freshness.json` 标记每字段更新时间
4. 限流次数 = 总请求数（无封禁告警）

### AC-2：增量更新
**Given** 已有缓存（最近一次更新为 T-1）
**When** T 日盘后触发增量更新
**Then**
1. 总耗时 ≤ 20 分钟
2. 仅拉取 T 日新增数据，不重复历史
3. 缓存命中率 > 95%

### AC-3：失败降级
**Given** 上游接口临时挂掉
**When** 触发拉取
**Then**
1. 重试 3 次（指数退避）
2. 重试失败后用缓存数据返回，标记 `is_stale=True`
3. 整体流程不崩溃
4. 失败原因写入日志 + 新鲜度报告

### AC-4：类型过滤
**Given** 调用方请求"仅 QDII + 主动股票基金"
**When** 返回基金列表
**Then**
1. 列表中不含债基、货基
2. 每只基金有 `fund_type` 字段标识

### AC-5：幂等性
**Given** 同一天已运行过一次
**When** 再次触发
**Then**
1. 不重复拉取已有数据
2. 缓存文件不变
3. 输出"今日已更新，跳过"日志

---

## 7. 边界与约束（显式不做）

- ❌ 不做实时盘中行情（只用 T+1 日频）
- ❌ 不做股票数据（仅公募基金）
- ❌ 不做债基、货基、REITs
- ❌ 不做付费数据源接入（Tushare 等留作 Roadmap）
- ❌ 不做代理池（1 req/s 单 IP 足够）
- ❌ 不做分布式（本地单进程）
