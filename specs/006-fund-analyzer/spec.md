# Feature #6 — fund-analyzer（单基金深度分析）

> 来源：[PRD §6 F3](../docs/prd.md) · [research.md §3](../docs/research.md)

---

## 1. Feature 简介

输入基金代码，触发 LLM 把结构化指标（来自 #4）翻译成 5 分钟可读懂的中文报告，含"建议/中性/回避"标签。**核心是防幻觉**（三层防御）。

---

## 2. 用户故事

- **作为用户**，我希望 5 分钟内看清楚某只基金值不值得买，每条结论都有数据支撑

---

## 3. 输入 / 输出

- 输入：基金代码
- 输出：中文 Markdown 报告（推送飞书卡片 + 写 Base）

---

## 4. 功能需求（FR）

### FR-1：报告生成
- 拉取该基金 L1-L4 全部指标（来自 #4）
- 用 quantstats 生成业绩归因补充数据
- LLM（DeepSeek-V3）把指标 JSON 翻译成中文报告
- 报告结构：① 一句话结论 + 标签 ② 经理画像 ③ 业绩归因 ④ 风格分析 ⑤ 现金流 ⑥ 风险提示 ⑦ 是否建议进观察池

### FR-2：防幻觉三层防御
1. **Prompt 层**：System Prompt 明确"唯一事实来源是 `<metrics>` 内 JSON，禁止编造，每条结论必须以【依据：指标名=数值】引用，证据不足输出'数据不足'"
2. **Schema 层**：LLM 用 JSON mode 输出报告结构（强制引用字段）
3. **代码层**：后校验（验证 LLM 引用的指标名真存在于输入 JSON；数值一致）

### FR-3：标签强制
- 报告结尾必须输出"建议/中性/回避"三选一
- 不输出标签 → 重试或降级到规则评级

### FR-4：触发方式
- 配置文件 `analyze: ["000001"]`
- CLI：`python -m src.analyzer 000001`
- 推送：渲染成卡片推送到飞书单聊

---

## 5. 非功能需求

| 维度 | 要求 |
|---|---|
| 性能 | 单基金分析 ≤ 30s（含 LLM 调用） |
| 成本 | 单基金 LLM token ≤ 3K input + 2K output |
| 准确性 | 后校验通过率 > 95%（不通过则重试） |
| 可解释 | 每条结论可追溯到指标 |

---

## 6. 验收标准（AC）

### AC-1：报告完整性
**Given** 000001
**When** 触发分析
**Then**
1. 5 分钟内生成 Markdown
2. 含 7 个章节
3. 结尾有"建议/中性/回避"标签

### AC-2：防幻觉
**Given** 报告输出
**When** 后校验
**Then**
1. 每条"【依据：...】"引用的指标真存在
2. 数值一致（误差 < 0.001）
3. 不通过率 < 5%

### AC-3：推送飞书
**Given** 报告生成
**When** 渲染卡片
**Then** 飞书单聊收到结构化卡片

## 7. 显式不做

- ❌ 不做多模型对比（MVP 单模型）
- ❌ 不做实时对话式分析（MVP 单次触发）
